用create_function绕过执行

关键点在于闭合符号执行命令

函数类似

create_function('$a,$b','return 111')

==>

function a($a, $b){
return 111;
}
  1. 如果可控在第一个参数，需要闭合圆括号和大括号：create_function(‘){}phpinfo();//‘, ‘’);
  2. 如果可控在第二个参数，需要闭合大括号：create_function(‘’, ‘}phpinfo();//‘);

结果

?action=\create_function&arg=11;}phpinfo();//

get了绕过正则的知识点

phpRCRE库使用NFA正则引擎库，即从起始状态开始，一个字符一个字符地读取输入串，并与正则表达式进行匹配，如果匹配不上，则进行回溯，尝试其他状态。

而PHP的pcre.backtrack_limit设定回溯次数最多为100万次

所以<?php phpinfo();//a*1000000即可绕过正则

https://regex101.com debug下，很清楚看到正则回溯流程

easy – phpmagic

<?php
if(isset($_GET['read-source'])) {
exit(show_source(__FILE__));
}

define('DATA_DIR', dirname(__FILE__) . '/data/' . md5($_SERVER['REMOTE_ADDR']));

if(!is_dir(DATA_DIR)) {
mkdir(DATA_DIR, 0755, true);
}
chdir(DATA_DIR);

$domain = isset($_POST['domain']) ? $_POST['domain'] : '';
$log_name = isset($_POST['log']) ? $_POST['log'] : date('-Y-m-d');
if(!empty($_POST) && $domain):
$command = sprintf("dig -t A -q %s", escapeshellarg($domain));
$output = shell_exec($command);

$output = htmlspecialchars($output, ENT_HTML401 | ENT_QUOTES);

$log_name = $_SERVER['SERVER_NAME'] . $log_name;
if(!in_array(pathinfo($log_name, PATHINFO_EXTENSION), ['php', 'php3', 'php4', 'php5', 'phtml', 'pht'], true)) {
file_put_contents($log_name, $output);
}

echo $output;
endif; 
?>

最后一步，log_name、output可控但是有过滤。

file_put_contents($filename, $data);

filename参数可以传伪协议流(有老哥说所有传filename参数的地方都可以传伪协议)，可以用伪协议绕过htmlspecialchars对文件内容的过滤。

文件名的拼接$_SERVER['SERVER_NAME']可以伪造host头

对后缀的过滤可以用/.绕过pathinfo的过滤。
